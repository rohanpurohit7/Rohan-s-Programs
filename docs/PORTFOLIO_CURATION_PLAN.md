## Repository Quality Standard

Each pinned repository should have:

```text
README.md
INDEX.md or docs/navigation file
src/ or notebooks/ with clear entry point
screenshots or SVG concept mockup
run instructions
.gitignore
CI workflow where code is buildable
privacy/redaction note for public artifacts
```

## Manual GitHub UI Actions

The connector does not expose profile pinning, repo archiving, or repo-topic editing. Complete these manually:

1. Go to GitHub profile.
2. Select **Customize your pins**.
3. Pin the recommended repositories above.
4. Archive or hide the low-signal repos listed above.
5. Add topics to strong repos, such as `javafx`, `data-engineering`, `spark`, `cybersecurity`, `cloud`, `devops`, `nosql`, `oracle-database`, `portfolio`.
6. Add demo screenshots or GIFs to Java and data-engineering repos.
